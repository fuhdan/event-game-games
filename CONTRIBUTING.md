# Contributing

Thank you for contributing to the Event Game Framework games repository.

## How to Contribute

Community contributions are welcome through pull requests.

Please use this repository for:
- new games
- improvements to existing games
- game documentation
- game-related fixes

Please do not include:
- backend changes
- frontend changes
- secrets, credentials, or private keys

## Contribution Process

1. Fork the repository
2. Create a branch for your change
3. Add or update your game
4. Open a pull request
5. Wait for review and feedback

## Game Structure

Each game should live in its own folder.

Example:

```text
games/my-new-game/
  README.md
  manifest.json
  assets/
  src/
```

## Requirements
Each game contribution should include:

- a clear folder name
- a README.md
- a manifest.json
- any required assets
- implementation files

## Pull Requests
Please:

- keep pull requests focused
- describe what your game does
- explain how it should be tested
- include screenshots if the game has a visual UI

## Code of Conduct
Be respectful and constructive in all interactions.
